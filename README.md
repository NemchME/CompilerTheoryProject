# CompilerTheoryProject

Проект реализует лексический и синтаксический анализатор подмножества языка Pascal.  
В результате разбора программы строится **абстрактное синтаксическое дерево (AST)**.

Также в проекте реализован **семантический анализ** AST.

Семантический анализ включает:

- проверку объявлений переменных
- проверку типов выражений
- проверку корректности условий (if / while)
- проверку функций и вызовов
- построение таблицы символов (scope)
- модификацию AST (вставка TypeConvert)

---

# Поддерживаемые конструкции языка

## Объявления

Поддерживается секция объявления переменных:

```
var
  x: integer;
```

Поддерживаемые типы:

- `integer`
- `char`
- `boolean`

---

## Операторы

Поддерживаются следующие операторы.

### Присваивание

```
x := expression
```

### Условный оператор

```
if condition then
  statement
else
  statement
```

### Цикл while

```
while condition do
  statement
```

### Цикл for

```
for i := a to b do
  statement
```

или

```
for i := a downto b do
  statement
```

### Управление циклом

```
break
continue
```

### Ввод / вывод

```
read(x)
readln(x)

write(x)
writeln(x)
```

# Семантический анализ

После построения AST выполняется семантический анализ программы.

## Таблица символов (Scope)

Используется структура `IdentScope`, которая хранит:

- переменные
- функции

Поддерживаются вложенные области видимости:

- глобальная область
- локальные области (if, while, block)
- параметры функций

Поиск переменных осуществляется вверх по цепочке scope.

---

## Проверка переменных

Реализованы проверки:

- использование необъявленной переменной → ошибка
- повторное объявление переменной → ошибка

---

## Проверка типов

Проверяются:

- арифметические операции → только integer
- логические операции → только boolean
- операции сравнения → возвращают boolean
- условия if / while → только boolean

---

## Функции

Поддерживаются:

- объявление функций
- параметры
- возврат значения (return)

Проверяется:

- существование функции
- количество аргументов
- типы аргументов

---

## Приведение типов

При несовпадении типов автоматически вставляется узел TypeConvert.

---


# Абстрактное синтаксическое дерево (AST)

После разбора программы строится AST.

В AST используются следующие узлы:

- `Program`
- `Block`
- `VarDecl`
- `CompoundStmt`
- `Assign`
- `If`
- `While`
- `For`
- `Break`
- `Continue`
- `Read`
- `Write`
- `BinOp`
- `UnOp`
- `Ident`
- `Literal`
- `Call`
- `TypeConvert`
- `Func`
- `Return`

Операции представлены перечислениями:

- `BinaryOpKind`
- `UnaryOpKind`

Это позволяет избежать строковых операторов и упрощает дальнейший анализ программы.

---

# Пример AST

Для программы:

```
program Demo;
var
  x: integer;
begin
  x := 2 + 3 * 4;
end.
```

строится дерево:

```
Program Demo
└─Block
  ├─VarDecl x : integer
  └─CompoundStmt
    └─Assign
      ├─Ident x
      └─BinOp +
        ├─Literal 2
        └─BinOp *
          ├─Literal 3
          └─Literal 4
```
